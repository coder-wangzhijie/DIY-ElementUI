- 目录
  - Tips
    - 最主要的是思路和功能实现，目录或者实现方式是其次。
    - 尽可能保持和源码一致
    - 为了保证思路，先按照自己的思路和习惯进行搭建
    - 最后再整理（封装、解耦、合并等等）
  - build：webpack 构建相关
    - webpack.dev.js
  - examples：（相当于 VueCli 模板下的 src，入口文件存放的地方）
    - entry.js
    - app.vue
    - assets
      - img
      - css
      - js
    - components
    - views/pages
  - packages：真正的组件开发目录
- 页面开发，进入最熟悉的页面开发
  - 页面结构搭建
    - header
      - 公用组件，注册为全局组件
    - content
      - VueRooter...
    - footer
      - 公用组件，注册为全局组件
